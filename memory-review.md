# Memory Review

Review the current memory/MEMORY.md for this project.
For each entry, evaluate:
- Still accurate? Does it reflect the current state of the project?
- Still relevant? Would this fact actually help in a future session?
- Duplicate? Does it overlap with another entry that covers it better?
- Too specific? Is it session-level detail that should be removed?
- 
Then:
- Propose a revised version of the full file with stale entries removed or merged
- Explain what you changed and why
- Do not write any changes until the user approves **important** changes to the memory file. If the user approves, write the revised memory file and explain the changes you made.
